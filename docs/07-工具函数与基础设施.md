## 1. Git 集成 (`src/utils/git/`)

### 1.1 gitFilesystem.ts — 免进程 Git 状态读取

核心思路：**直接读文件系统，避免 spawn git 子进程**，显著减少 IPC 开销。

```
resolveGitDir(startPath?)
  → 解析 .git 目录（支持 worktree/submodule）
  → worktree 时 .git 是文件，内容 "gitdir: <path>"
  → 结果缓存在 Map 中
```

关键能力：
- **HEAD 解析**：读 `.git/HEAD`，格式 `ref: refs/heads/<branch>` 或裸 SHA
- **Ref 解析**：先查 loose file (`refs/heads/xxx`)，再查 `packed-refs`
- **GitHeadWatcher**：用 `fs.watchFile` 监听 HEAD 变化，缓存分支名和 SHA
- **浅克隆检测**：检查 `<commonDir>/shallow` 文件是否存在

### 1.2 gitConfigParser.ts — .git/config 解析器

纯 TypeScript 实现，遵循 git `config.c` 规范：
- Section name 大小写不敏感，subsection（引号内）大小写敏感
- 支持反斜杠转义、行内注释（`#` 或 `;`）

### 1.3 gitignore.ts — gitignore 操作

- `isPathGitignored()` — 调用 `git check-ignore`，exit 0 = 被忽略
- `addFileGlobRuleToGitignore()` — 向全局 gitignore (`~/.config/git/ignore`) 追加规则

### 1.4 versions.ts — Git 仓库状态检查

```typescript
projectIsInGitRepo(cwd) → boolean  // 同步，用 findGitRoot 遍历文件系统
```

## 2. Bash 命令执行 (`src/utils/bash/`)

### 2.1 bashParser.ts — 纯 TS Bash 解析器

**不依赖 WASM/native**，纯 TypeScript 实现 tree-sitter-bash 兼容 AST。

设计要点：
- 输出 `TsNode` 树，含 `type / text / startIndex / endIndex / children`
- **50ms 超时**防止恶意/病态输入（可传 `Infinity` 禁用）
- **50,000 节点上限**防 OOM
- 完整词法分析：单引号、双引号、ANSI-C 引号、`$()`、`${}`、反引号等

### 2.2 ast.ts — 安全分析（Fail-Closed 设计）

核心原则：**显式白名单 + 不认识就拒绝**。

```
parseForSecurity(command) →
  | { kind: 'simple', commands: SimpleCommand[] }   // 可信 argv
  | { kind: 'too-complex', reason: string }          // 需用户确认
  | { kind: 'parse-unavailable' }
```

- 遇到白名单外的 AST 节点类型 → `too-complex` → 走权限提示流程
- 提取每个简单命令的 `argv[]`、环境变量赋值、重定向
- 命令替换 `$()` 用 `__CMDSUB_OUTPUT__` 占位，内部命令单独检查权限

### 2.3 commands.ts — 命令解析与安全过滤

**注入防护**：占位符使用随机盐（`randomBytes(8)`），防止恶意命令包含字面占位符字符串。

重定向安全检查 `isStaticRedirectTarget()`，拒绝：
- 含空白/引号（防解析差异攻击）
- `$` 变量、反引号、glob、花括号展开、波浪号、进程替换
- `#` 开头（shell-quote 解析差异）、`!` 历史展开、`=` Zsh 等号展开

续行处理：奇数反斜杠 + 换行 = 续行；偶数反斜杠 = 命令分隔符。防止 `tr\<newline>aceroute` 绕过。

### 2.4 registry.ts — 命令规格注册

```typescript
CommandSpec = { name, subcommands?, args?, options? }
Argument = { isDangerous?, isVariadic?, isCommand?, isModule?, isScript? }
```

- 内置规格 + `@withfig/autocomplete` 动态加载（LRU 缓存）
- `isCommand: true` 标记包装器命令（如 `timeout`、`sudo`）

### 2.5 shellPrefix.ts — Shell 前缀格式化

处理带路径和参数的 shell 前缀，正确引用含空格路径。

## 4. MCP 协议工具 (`src/utils/mcp/`)

### 4.1 dateTimeParser.ts — 自然语言日期解析

**用 Haiku 模型做 NL→ISO 8601 转换**：

```
"tomorrow at 3pm" → "2025-10-15T15:00:00-07:00"
"next Monday"     → "2025-10-20"
```

流程：构造含时区/星期上下文的 prompt → 调用 `queryHaiku()` → 校验输出以 `\d{4}` 开头。

`looksLikeISO8601()` 先判断输入是否已是 ISO 格式，避免不必要的 API 调用。

### 4.2 elicitationValidation.ts — MCP 表单校验

基于 Zod 的 schema 验证，支持 MCP Elicitation 规范中的所有类型：

- **string**：minLength/maxLength、format（email/uri/date/date-time）
- **number/integer**：min/max 范围，整数约束
- **boolean**：强制转换
- **enum**：单选（`oneOf` + `enum`）、多选（`array.items.anyOf`）

日期时间字段：同步校验失败 → 异步尝试 Haiku NL 解析 → 再次校验。

## 6. 常量定义 (`src/constants/`)

### 6.1 toolLimits.ts — 工具结果大小限制

| 常量 | 值 | 说明 |
|------|-----|------|
| `DEFAULT_MAX_RESULT_SIZE_CHARS` | 50,000 | 单工具结果最大字符数 |
| `MAX_TOOL_RESULT_TOKENS` | 100,000 | 单结果最大 token 数 |
| `MAX_TOOL_RESULT_BYTES` | 400KB | 按 4 bytes/token 换算 |
| `MAX_TOOL_RESULTS_PER_MESSAGE_CHARS` | 200,000 | 单条消息所有工具结果总上限 |

超限处理：持久化到磁盘，向模型返回预览 + 文件路径。

### 6.2 apiLimits.ts — API 限制

- 图片：base64 最大 5MB，客户端缩放到 2000px
- PDF：原始最大 20MB，单次最多读 20 页，超 3MB 走页面提取
- 媒体：单请求最多 100 个媒体项

### 6.3 tools.ts — 工具可用性矩阵

```
ALL_AGENT_DISALLOWED_TOOLS  — Agent 禁用工具（防递归）
ASYNC_AGENT_ALLOWED_TOOLS   — 异步 Agent 可用工具
IN_PROCESS_TEAMMATE_ALLOWED — 进程内队友额外工具（TaskCreate/Get/List/Update/SendMessage）
COORDINATOR_MODE_ALLOWED    — 协调器模式仅允许 Agent/TaskStop/SendMessage
```

### 6.4 files.ts — 二进制文件检测

- `BINARY_EXTENSIONS`：约 100 种扩展名（图片/视频/音频/归档/可执行/字体/数据库等）
- `isBinaryContent(buffer)`：检查前 8KB，有 null 字节或 >10% 不可打印字符即判定为二进制

### 6.5 common.ts — 通用常量

- `getLocalISODate()` / `getSessionStartDate()`：会话级日期缓存，避免 prompt cache 在午夜失效
- `getLocalMonthYear()`：月粒度日期（用于工具 prompt，减少 cache busting）

## 7. Hooks Schema (`src/schemas/hooks.ts`)

用 Zod v4 定义 Hook 配置校验，支持 4 种 Hook 类型：

| 类型 | 说明 | 关键字段 |
|------|------|----------|
| `command` | Shell 命令 | `command`, `shell`(bash/powershell), `async`, `once` |
| `prompt` | LLM Prompt | `prompt`($ARGUMENTS 占位), `model` |
| `agent` | Agent 验证器 | `prompt`, `model`(默认 Haiku) |
| `http` | HTTP POST | `url`, `headers`(支持 `$VAR` 环境变量插值), `allowedEnvVars` |

通用字段：
- `if`：权限规则语法过滤（如 `"Bash(git *)"`)，避免无关 hook 启动
- `timeout`：超时秒数
- `statusMessage`：spinner 显示文本
- `once`：执行一次后移除

Hook 事件配置：`{ [HookEvent]: HookMatcher[] }`，每个 Matcher 含 `matcher`（模式匹配）+ `hooks[]`（执行列表）。

## 架构要点总结

1. **性能优先**：Git 状态直接读文件系统，Bash 解析器纯 TS 无 WASM，日期缓存防 cache bust
2. **安全纵深**：Bash 分析 fail-closed 设计，随机盐防注入，重定向目标严格白名单
3. **分层解耦**：沙箱通过 adapter 桥接外部包，MCP 校验独立于协议层，Hook schema 从 settings 抽离防循环依赖
4. **渐进增强**：TeamMem 通过 feature flag 控制，命令规格支持 fig 动态加载
