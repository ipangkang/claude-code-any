# 任务、技能与插件系统

## 一、Task 系统 (src/tasks/)

Task 是 Claude Code 的后台任务抽象。每个 Task 是一个独立的执行单元，可在后台运行。

### Task 类型

| 类型 | 文件 | 用途 |
|------|------|------|
| `LocalShellTask` | LocalShellTask.tsx | 本地 shell 命令后台执行 |
| `LocalAgentTask` | LocalAgentTask.tsx | 本地子 Agent（AgentTool 创建）|
| `RemoteAgentTask` | RemoteAgentTask.tsx | 远程 Agent（通过 API 执行）|
| `InProcessTeammateTask` | InProcessTeammateTask.tsx | 进程内 Teammate（swarm 模式）|
| `DreamTask` | DreamTask.ts | 后台记忆整理任务 |
| `LocalWorkflowTask` | (feature-gated) | 工作流脚本任务 |
| `MonitorMcpTask` | (feature-gated) | MCP 监控任务 |

### Task 状态机

```
pending → running → completed
                  → failed
                  → cancelled (通过 TaskStopTool)
```

### 前台/后台判定

```typescript
function isBackgroundTask(task: TaskState): boolean {
  // 必须是 running 或 pending
  // 且 isBackgrounded !== false（前台任务不算后台任务）
}
```

Task 通过 `TaskCreateTool` 创建，`TaskStopTool` 停止，`TaskOutputTool` 读取输出，`TaskListTool` 列出所有任务。

### LocalAgentTask — 子 Agent 核心

这是 `AgentTool` 的底层实现。创建子 Agent 时：
1. 创建新的 QueryEngine 实例（继承工具集但限制可用工具）
2. 子 Agent 有独立的消息历史和上下文
3. 通过 `ASYNC_AGENT_ALLOWED_TOOLS` 白名单限制工具访问
4. 支持 `isolation: "worktree"` 在 git worktree 中运行

## 二、Skill 系统 (src/skills/)

Skill 是可复用的 prompt 模板，通过 `/skill-name` 或 `SkillTool` 调用。

### 加载机制 (loadSkillsDir.ts)

Skills 从多个来源加载，按优先级排序：

```
1. bundled（内置）   → src/skills/bundled/
2. managed（策略）   → 管理设置路径/.claude/skills/
3. userSettings     → ~/.claude/skills/
4. projectSettings  → .claude/skills/
5. plugin           → 插件注册
6. mcp              → MCP 服务器提供
```

### Skill 文件格式

```markdown
---
name: skill-name
description: 简要描述
whenToUse: 触发条件描述
model: claude-sonnet-4-6  # 可选指定模型
shell: bash               # 可选 shell 命令
tools: [Bash, Read, Edit] # 可选工具白名单
---

Skill 的 prompt 内容（markdown）...
```

### 内置 Skills

| Skill | 用途 |
|-------|------|
| `loop` | 定时循环执行 prompt |
| `claudeApi` | Claude API 使用指导 |
| `debug` | 系统化调试流程 |
| `remember` | 记忆保存 |
| `simplify` | 代码精简审查 |
| `verify` | 验证完成度 |
| `batch` | 批量操作 |
| `scheduleRemoteAgents` | 远程 Agent 调度 |
| `updateConfig` | 配置更新指导 |
| `skillify` | 创建新 Skill |

### 执行流程

```
用户输入 /skill-name 或 SkillTool 调用
  → 查找匹配的 Skill 定义
  → 解析 frontmatter 配置
  → 替换参数占位符
  → 执行 shell 命令（如有）
  → 返回 prompt 内容给模型
```

## 三、Plugin 系统 (src/plugins/)

Plugin 是扩展 Claude Code 功能的机制。

### 内置插件 (builtinPlugins.ts)

内置插件提供额外的斜杠命令和 Skill。加载方式：
- `getBuiltinPluginSkillCommands()` → 获取插件提供的 Skill 命令
- 通过 `utils/plugins/loadPluginCommands.ts` 动态加载

### 插件加载 (utils/plugins/)

```
pluginLoader.ts → 从 .claude/plugins/ 和配置中加载插件定义
loadPluginCommands.ts → 将插件转换为 Command 对象
  - getPluginCommands() → 获取插件命令
  - getPluginSkills() → 获取插件技能
```

### 插件与 Skill 的关系

Plugin 可以注册：
- 斜杠命令（`Command` 类型）
- Skills（`PromptCommand` 类型）
- 两者都可以提供 `tools` 白名单限制可用工具

## 四、Coordinator 系统 (src/coordinator/)

Coordinator 模式是多 Agent 协作的高级功能（feature-gated: `COORDINATOR_MODE`）。

### 核心机制

```typescript
function isCoordinatorMode(): boolean {
  return isEnvTruthy(process.env.CLAUDE_CODE_COORDINATOR_MODE)
}
```

启用后：
- **Coordinator Agent**（主控）只能使用 AgentTool 和 TaskStopTool
- **Worker Agent**（执行者）使用 `ASYNC_AGENT_ALLOWED_TOOLS` 白名单中的工具
- Worker 工具集：Bash, FileRead, FileEdit, FileWrite, Glob, Grep, WebFetch 等

### 协调上下文

`getCoordinatorUserContext()` 为协调器注入额外上下文：
- 可用 Worker 工具列表
- MCP 服务器信息
- Scratchpad 目录路径（用于 Agent 间共享数据）

### 会话模式匹配

`matchSessionMode()` 确保恢复会话时协调器模式一致：
- 记录会话创建时的模式（`coordinator` 或 `normal`）
- 恢复时自动切换环境变量匹配原始模式

## 五、系统间协作关系

```
用户请求
  │
  ├─ /skill-name → Skill 系统 → 生成 prompt → QueryEngine
  │
  ├─ AgentTool → Task 系统 → 创建 LocalAgentTask
  │                         → 新 QueryEngine 实例
  │                         → 独立上下文执行
  │
  ├─ Coordinator 模式:
  │   主控 QueryEngine
  │     ├─ AgentTool → Worker 1 (LocalAgentTask)
  │     ├─ AgentTool → Worker 2 (LocalAgentTask)
  │     └─ SendMessage → 向 Worker 发送消息
  │
  └─ Plugin → 注册额外 Skill / Command
```

### 关键设计原则

1. **隔离性**: 每个子 Agent 有独立的 QueryEngine、消息历史和文件缓存
2. **工具白名单**: 子 Agent 只能访问 `ASYNC_AGENT_ALLOWED_TOOLS` 中的工具
3. **可组合性**: Skill 可以触发 Tool 调用，Tool 可以创建 Task，Task 可以运行新的 QueryEngine
4. **Feature-gated**: 高级功能（Coordinator、Workflow、Monitor）通过编译时 flag 控制
