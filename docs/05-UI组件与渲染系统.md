# 05 - UI 组件与渲染系统

Claude Code 的 UI 基于 **Ink** (React 终端渲染框架) 构建，用 Yoga Flexbox 在终端中实现组件化交互。

---

## 一、Ink 渲染引擎 (`src/ink/`)

### 1.1 渲染管线

Ink 是自定义 React reconciler，将组件树渲染为终端 ANSI 输出：

```
React 组件树 → reconciler.ts → 虚拟 DOM (dom.ts) → Yoga 布局 → Screen 缓冲 → 终端差分输出
```

| 模块 | 职责 |
|------|------|
| `reconciler.ts` | 基于 `react-reconciler`，实现 createNode/appendChild 等 DOM 操作 |
| `ink.tsx` | Ink 类：渲染循环、事件分发、鼠标选择、搜索高亮 |
| `renderer.ts` | Yoga 计算布局后生成 Output |
| `screen.ts` | 双缓冲屏幕，Cell 级别样式/宽度/超链接池化 |
| `render-to-screen.ts` | Screen 前后帧差分，只输出变化部分 |
| `layout/yoga.ts` | Ink 样式属性 -> Yoga Edge/Align/FlexDirection 映射 |

### 1.2 事件系统 (`src/ink/events/`)

- `input-event.ts`：键盘输入（Key 含 ctrl/shift/meta/arrow 标志位）
- `click-event.ts`：鼠标点击（仅 AlternateScreen 模式）
- `keyboard-event.ts`：DOM 级键盘事件，支持 stopImmediatePropagation
- `dispatcher.ts` / `emitter.ts`：事件分发与冒泡

### 1.3 核心 Hooks 与组件

| Hook | 用途 |
|------|------|
| `use-input.ts` | 键盘输入处理，设置 raw mode |
| `use-selection.ts` | Alt Screen 模式下的鼠标拖选 |
| `use-search-highlight.ts` | 搜索高亮 |
| `use-terminal-viewport.ts` | 视口尺寸订阅 |

基础组件：`Box`(Flexbox 容器)、`Text`(文本)、`ScrollBox`(可滚动区域)、`AlternateScreen`(全屏)

## 二、屏幕与布局 (`src/screens/`)

### 2.1 三个屏幕

| 屏幕 | 文件 | 功能 |
|------|------|------|
| REPL | `REPL.tsx` (5000+ 行) | 主交互界面 |
| Doctor | `Doctor.tsx` | 诊断工具 (`claude doctor`) |
| ResumeConversation | `ResumeConversation.tsx` | 恢复历史会话 |

### 2.2 REPL 组件层次

```
App (FpsMetrics + Stats + AppState Provider)
  └─ KeybindingSetup
       └─ FullscreenLayout
            ├─ scrollable: Messages
            │    ├─ LogoHeader
            │    ├─ MessageRow[] (UserPrompt/AssistantText/ToolUse/ToolResult...)
            │    └─ Spinner
            ├─ overlay: PermissionRequest (权限弹窗)
            ├─ modal: 各种对话框 (Help/History/Export...)
            └─ bottom: PromptInput (输入框)
```

### 2.3 FullscreenLayout

`FullscreenLayout.tsx` 切分四个区域：
- **scrollable**：ScrollBox 包裹的消息列表，支持虚拟滚动
- **bottom**：固定底部输入框
- **overlay**：ScrollBox 内部的覆盖层（权限对话框）
- **modal**：绝对定位弹窗，覆盖全屏
- 附加：unseen pill ("N new messages")、sticky prompt header

## 三、用户输入 (`src/components/PromptInput/`)

### 3.1 子模块

| 子模块 | 职责 |
|--------|------|
| `inputModes.ts` | 输入模式（普通/Bash `!`/Plan `#`，前缀字符切换） |
| `inputPaste.ts` | 粘贴处理（长文本/图片） |
| `PromptInputFooter.tsx` | 底部状态栏（模型/Token/快捷键） |
| `PromptInputModeIndicator.tsx` | 当前模式指示 |
| `Notifications.tsx` | 临时通知 |

### 3.2 输入处理流程

```
按键 → useInput (raw mode) → TextInput/VimTextInput
  → inputModes 解析前缀 → useArrowKeyHistory (历史翻阅)
    → useTypeahead (补全) → usePromptSuggestion (AI 建议)
      → handlePromptSubmit (提交)
```

关键特性：Vim 模式、图片粘贴、Slash 命令补全、快捷键绑定 (`useKeybinding`)、模型切换

## 四、消息展示 (`src/components/messages/`)

### 4.1 消息类型 -> 组件映射

**用户侧：** `UserPromptMessage`、`UserBashInputMessage`/`UserBashOutputMessage`、`UserImageMessage`、`UserCommandMessage`

**助手侧：** `AssistantTextMessage`(含 Markdown)、`AssistantThinkingMessage`(思考过程)、`AssistantToolUseMessage`(工具调用 + 进度)

**工具结果：** `UserToolResultMessage/` 子目录 — Success/Error/Canceled/Rejected

**系统：** `SystemTextMessage`、`RateLimitMessage`、`CompactBoundaryMessage`(压缩边界)

### 4.2 Messages 容器 (`Messages.tsx`)

- 消息规范化 (`normalizeMessages`) 和 UI 重排序
- 工具调用分组 (`applyGrouping`、`collapseReadSearchGroups`)
- 虚拟列表渲染 (`VirtualMessageList`)
- LogoHeader 用 React.memo 隔离，防止 messages 变化级联重绘

## 五、权限对话框 (`src/components/permissions/`)

### 5.1 路由分发

`PermissionRequest.tsx` 中 `permissionComponentForTool(tool)` 按工具类型 switch：

```
BashTool       → BashPermissionRequest
FileEditTool   → FileEditPermissionRequest
FileWriteTool  → FileWritePermissionRequest
WebFetchTool   → WebFetchPermissionRequest
NotebookEditTool → NotebookEditPermissionRequest
SkillTool      → SkillPermissionRequest
AskUserQuestion → AskUserQuestionPermissionRequest
其他           → FallbackPermissionRequest
```

### 5.2 PermissionDialog 基础壳

所有权限对话框共用：圆角上边框、标题 + WorkerBadge(多 Agent)、内容区 + 选项列表。

### 5.3 交互流程

```
Claude 请求工具 → 权限引擎 (PermissionDecision)
  ├─ 自动放行 → 直接执行
  └─ 需审批 → 显示 PermissionRequest
       ├─ Allow (y)      → onDone()
       ├─ Deny (n)       → onReject()
       ├─ Always Allow   → 持久化 PermissionUpdate 规则
       └─ 分类器自动审批  → shimmer 动画等待异步检查
```

### 5.4 Bash 权限的特殊处理

- 命令前缀解析 + 描述性规则生成
- sed 命令自动切换到 `SedEditPermissionRequest`
- 分类器自动审批（`ClassifierCheckingSubtitle` shimmer 动画）
- 破坏性命令警告、沙箱模式支持

### 5.5 规则管理 (`permissions/rules/`)

`AddPermissionRules` / `PermissionRuleList` / `RecentDenialsTab` / `WorkspaceTab`

## 六、性能优化

1. **双缓冲差分**：Screen 前后帧比较，只输出变化 Cell
2. **React Compiler**：`_c()` memo cache 自动跳过不变子树
3. **对象池化**：StylePool / CharPool / HyperlinkPool 减少 GC
4. **虚拟化**：VirtualMessageList 只渲染可视区域
5. **OffscreenFreeze**：冻结不可见区域
6. **帧率控制**：`FRAME_INTERVAL_MS` 限制渲染频率

---

## 七、设计模式总结

- **Provider 嵌套**：App → FpsMetrics → Stats → AppState，状态自顶向下
- **类型分发**：消息和权限对话框都通过 switch 按类型选择组件
- **Hook 组合**：输入处理由 10+ Hook 分层组合
- **类 DOM 事件冒泡**：Ink 支持 stopImmediatePropagation
- **Yoga Flexbox**：终端布局由 Yoga 引擎计算，组件只声明样式
