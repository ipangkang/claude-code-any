# 12 - 构建脚本与外部模块

> Claude Code 顶层非 src/ 目录解析：构建流水线、存根系统、外部工具、类型声明、工具函数、第三方原生模块。

---

## 1. scripts/ — 构建脚本

Claude Code 原生使用 **Bun** 构建，但源码仓库提供了基于 **esbuild** 的替代构建方案。

### 1.1 build.mjs — 主构建入口

核心流程分四个阶段：

| 阶段 | 动作 |
|------|------|
| Phase 1 | 复制 `src/` 到 `build-src/`，保持原始源码不变 |
| Phase 2 | 源码变换：`feature('X')` → `false`，`MACRO.X` → 字面量，移除 `bun:bundle` 导入 |
| Phase 3 | 创建入口包装器 `build-src/entry.ts`，导入 `src/entrypoints/cli.tsx` |
| Phase 4 | 迭代式 esbuild 打包（最多 5 轮），每轮自动为缺失模块创建存根 |

关键设计：
- **MACRO 替换表**：将 `MACRO.VERSION`、`MACRO.FEEDBACK_CHANNEL` 等编译时常量替换为字符串字面量
- **迭代容错**：每轮打包失败后解析 `Could not resolve "X"` 错误，自动生成存根，再重试
- 输出文件：`dist/cli.js`（ESM 格式，Node 18+）

### 1.2 prepare-src.mjs — 预处理脚本

在源码树**原地**进行变换（与 build.mjs 的"复制后变换"不同）：
- 替换 `bun:bundle` 导入为相对路径指向 `stubs/bun-bundle.js`
- 根据文件在目录树中的深度自动计算正确的相对路径
- 创建 `stubs/bun-ffi.ts`（`bun:ffi` 的空存根）
- 重写 `stubs/global.d.ts` 声明 MACRO 类型

### 1.3 transform.mjs — 替代构建脚本

与 build.mjs 功能相似，但采用不同策略：
- 通过 `globalThis.MACRO = {...}` 在入口文件注入全局变量（而非文本替换）
- 仅处理 `bun:bundle` 导入替换，不做 `feature()` 替换
- 构建失败时给出手动修复指引

### 1.4 stub-modules.mjs — 自动存根生成器

独立运行的存根创建工具：
1. 先执行一次 esbuild 打包，收集所有 `Could not resolve` 错误
2. 用 `grep` 定位每个缺失模块的导入位置
3. 根据文件类型生成存根：`.d.ts` → 空类型导出，`.txt/.md` → 空文件，`.ts/.js` → 空函数导出
4. 存根创建后自动重试打包

## 2. stubs/ — 存根文件

为脱离 Bun 运行时而设计的编译时替代品。

| 文件 | 用途 |
|------|------|
| `bun-bundle.ts` | 替代 `bun:bundle` 模块，`feature()` 固定返回 `false`（禁用所有特性门控） |
| `global.d.ts` | 声明全局 `MACRO` 对象的 TypeScript 类型（VERSION, BUILD_TIME 等 7 个字段） |
| `macros.d.ts` | 同上，另一份 MACRO 类型声明（内容略有不同） |
| `macros.ts` | 通过 `declare global` 扩展全局作用域声明 MACRO，含 `ISSUES_EXPLAINER_URL` 和 `FEEDBACK_CHANNEL_URL` 两个额外字段 |

核心机制：`feature()` 始终返回 `false`，意味着所有被 `feature('X')` 保护的内部/实验性功能在源码构建中均被禁用。

## 3. tools/ — 外部工具存根

这五个目录均为**自动生成的空存根**，对应内部功能门控模块：

| 工具 | 文件 | 推测用途 |
|------|------|----------|
| OverflowTestTool | `OverflowTestTool.js` | 溢出/压力测试工具 |
| TerminalCaptureTool | `prompt.js` | 终端截屏/输出捕获的提示词 |
| TungstenTool | `TungstenTool.js` | Tungsten 内部服务集成工具 |
| VerifyPlanExecutionTool | `constants.js` | 计划执行验证的常量定义 |
| WorkflowTool | `constants.js` | 工作流工具的常量定义 |

所有文件内容一致：`export default function X() {}; export const X = () => {}`。这些是通过 `feature()` 门控的内部专属工具，在开源版中不可用。

## 4. types/ — 顶层类型声明

仅一个文件：

- **`connectorText.js`** — 自动生成的空存根，`export default function connectorText() {}`

推测为连接器文本格式化相关的类型定义，在开源版中被存根替代。

## 5. utils/ — 顶层工具函数

三个文件均为自动生成的空存根：

| 文件 | 推测用途 |
|------|----------|
| `attributionHooks.js` | 归因/来源追踪钩子（可能用于内部 telemetry） |
| `systemThemeWatcher.js` | 系统主题（深色/浅色）监听器 |
| `udsClient.js` | Unix Domain Socket 客户端（进程间通信） |

这些工具函数的实际实现被 `feature()` 保护，开源版无法访问。

## 6. vendor/ — 第三方原生模块

四个平台原生（N-API）模块的 TypeScript 包装层，使用**懒加载**模式统一管理 `.node` 二进制文件。

### 6.1 audio-capture-src — 音频采集

- **平台**：macOS、Linux、Windows
- **功能**：录音（startRecording/stopRecording）、播放（startPlayback/writePlaybackData/stopPlayback）、麦克风权限查询
- **权限状态**：macOS 使用 TCC 框架（0=未确定, 1=受限, 2=拒绝, 3=已授权），Linux 始终返回 3，Windows 检查注册表
- **加载优先级**：`AUDIO_CAPTURE_NODE_PATH` 环境变量 → `vendor/audio-capture/{arch}-{platform}/` → 相对路径回退

### 6.2 image-processor-src — 图像处理

- **功能**：实现了兼容 **sharp** API 的图像处理链（resize, jpeg, png, webp, toBuffer）
- **额外能力**：剪贴板图像读取（`readClipboardImage`，仅 macOS，通过 CoreGraphics/ImageIO）
- **设计模式**：链式调用 + 延迟执行，操作队列在 `toBuffer()` 时才实际应用
- **导出**：`sharp()` 工厂函数 + `getNativeModule()` 原始绑定访问器

### 6.3 modifiers-napi-src — 键盘修饰键检测

- **平台**：仅 macOS
- **功能**：`getModifiers()` 获取当前按下的修饰键列表，`isModifierPressed(key)` 查询特定键状态
- **预热**：提供 `prewarm()` 函数，在启动时提前加载原生模块避免首次使用延迟
- **加载路径**：`MODIFIERS_NODE_PATH` 环境变量（打包模式） → `vendor/modifiers-napi/{arch}-darwin/modifiers.node`（开发模式）

### 6.4 url-handler-src — URL 事件处理

- **平台**：仅 macOS
- **功能**：`waitForUrlEvent(timeoutMs)` 监听 macOS Apple Event（`kAEGetURL`）
- **机制**：初始化 NSApplication，注册 URL 事件处理器，在指定超时内泵送事件循环
- **用途**：OAuth 回调等需要从浏览器跳转回 CLI 的场景

### vendor 模块共同模式

```
懒加载 → 缓存 → 环境变量路径优先 → 回退到 vendor 目录 → 失败返回 null
```

所有模块都遵循"优雅降级"：原生模块不可用时返回 null/false/空数组，不抛异常。

## 总结

| 目录 | 性质 | 核心作用 |
|------|------|----------|
| scripts/ | 构建基础设施 | 将 Bun 专属源码转译为 esbuild 可构建的形式 |
| stubs/ | 编译时替代 | 替换 `bun:bundle` 和 MACRO 编译时内联机制 |
| tools/ | 功能存根 | 内部工具的空壳，通过 feature() 门控 |
| types/ | 类型存根 | 内部类型定义的空壳 |
| utils/ | 工具存根 | 内部工具函数的空壳 |
| vendor/ | 原生桥接 | 音频、图像、键盘、URL 四个 N-API 原生模块的 TS 包装层 |

整体设计思路：**通过 `feature()` 编译时门控 + 存根替代 + 懒加载降级**，实现同一套源码在内部 Bun 构建和外部 esbuild 构建下的兼容运行。
