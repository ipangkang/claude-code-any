## 1. src/query/ -- 查询子模块

查询子模块将 `query()` 函数的关注点拆分为独立文件，实现配置快照、依赖注入、停止钩子和 Token 预算控制。

### 1.1 config.ts -- 查询配置快照

在每次 `query()` 调用入口冻结一份不可变配置，使后续步骤可作为纯函数 `(state, event, config)` 处理。

```ts
type QueryConfig = {
  sessionId: SessionId
  gates: {
    streamingToolExecution: boolean  // 流式工具执行
    emitToolUseSummaries: boolean    // 工具摘要输出
    isAnt: boolean                  // 内部用户标记
    fastModeEnabled: boolean        // 快速模式开关
  }
}
```

- **关键函数**: `buildQueryConfig()` -- 读取 Statsig 缓存门控和环境变量，一次快照
- **设计要点**: 故意排除 `feature()` 门控（用于 tree-shaking），仅包含运行时门控

### 1.2 deps.ts -- 依赖注入

将 `query()` 的 I/O 依赖收拢为 `QueryDeps` 接口，测试时可直接注入 fake 替代 spy。

```ts
type QueryDeps = {
  callModel: typeof queryModelWithStreaming   // 模型调用
  microcompact: typeof microcompactMessages  // 微压缩
  autocompact: typeof autoCompactIfNeeded    // 自动压缩
  uuid: () => string                         // UUID 生成
}
```

- **关键函数**: `productionDeps()` -- 返回真实依赖的工厂函数
- **设计要点**: 用 `typeof fn` 保持签名同步，范围故意窄（4 个依赖）以验证模式

### 1.3 tokenBudget.ts -- Token 预算控制

跟踪当前轮次 Token 消耗，决定是否继续生成或停止。

```ts
type BudgetTracker = {
  continuationCount: number       // 已续写次数
  lastDeltaTokens: number         // 上次增量
  lastGlobalTurnTokens: number    // 上次全局 Token 数
  startedAt: number               // 开始时间戳
}
type TokenBudgetDecision = ContinueDecision | StopDecision
```

- **关键函数**: `checkTokenBudget(tracker, agentId, budget, globalTurnTokens)`
- **核心逻辑**:
  - 子代理（agentId 存在）或无预算时直接停止
  - 消耗 < 90% 预算 → 继续（发送 nudge 消息）
  - 连续 3+ 次增量 < 500 Token → 收益递减，停止
  - 返回结构化决策供外部处理

### 1.4 stopHooks.ts -- 停止钩子处理

`handleStopHooks()` 是一个大型异步生成器，在模型完成一轮回答后执行一系列后处理。

**职责链**:
1. 保存缓存安全参数（供 `/btw` 和 SDK 使用）
2. 模板任务分类（若运行在 Job 模式下）
3. 非 bare 模式下触发后台任务：提示建议、记忆提取、自动梦境
4. 计算机使用清理（CHICAGO_MCP 门控）
5. 执行 Stop 钩子 → 收集阻塞错误 / 阻止续写
6. 若为 Teammate：执行 TaskCompleted 和 TeammateIdle 钩子

**关键接口**:
```ts
function handleStopHooks(...): AsyncGenerator<StreamEvent | Message, StopHookResult>
type StopHookResult = { blockingErrors: Message[]; preventContinuation: boolean }
```

## 2. src/context/ -- React 上下文管理

全部文件为 React Context Provider + Hook 模式（已经过 React Compiler 编译），为 TUI 组件树提供共享状态。

| 文件 | Context | 用途 |
|------|---------|------|
| `QueuedMessageContext.tsx` | `QueuedMessageContext` | 标记消息是否在队列中、是否首条、容器内边距 |
| `fpsMetrics.tsx` | `FpsMetricsContext` | 提供 FPS 指标 getter 函数 |
| `mailbox.tsx` | `MailboxContext` | 全局信箱实例（进程内消息传递） |
| `modalContext.tsx` | `ModalContext` | 模态对话框尺寸与滚动引用 |
| `notifications.tsx` | 通过 AppState | 通知队列管理（优先级、折叠、超时） |
| `overlayContext.tsx` | 通过 AppState | 覆盖层注册（Escape 键协调） |
| `promptOverlayContext.tsx` | `DataContext` / `SetContext` 等 | 浮动提示覆盖层（建议、对话框） |
| `stats.tsx` | `StatsContext` | 指标收集（计数器、直方图、集合），进程退出时持久化 |
| `voice.tsx` | `VoiceContext` | 语音输入状态（录音/处理/空闲） |

**共同模式**:
- Provider 创建单例 store，通过 `useSyncExternalStore` 订阅切片避免不必要渲染
- `notifications.tsx` 实现了完整的优先级队列：`immediate > high > medium > low`
- `stats.tsx` 的 `StatsStore` 使用蓄水池采样（Algorithm R）记录直方图

## 3. src/types/ -- 类型定义系统

### 3.1 ids.ts -- 品牌类型

用 TypeScript 品牌类型（Branded Types）在编译期区分 SessionId 和 AgentId：

```ts
type SessionId = string & { readonly __brand: 'SessionId' }
type AgentId   = string & { readonly __brand: 'AgentId' }
```

- `toAgentId(s)` 校验格式 `a[label-]<16 hex chars>` 后品牌化
- 防止意外混用会话 ID 和代理 ID

### 3.2 command.ts -- 命令系统类型

定义斜杠命令的完整类型体系：

| 类型 | 说明 |
|------|------|
| `PromptCommand` | 发送至模型的提示型命令（含技能钩子、fork 上下文） |
| `LocalCommand` | 本地执行的 JS 命令（懒加载） |
| `LocalJSXCommand` | 返回 JSX 节点的本地命令 |
| `CommandAvailability` | 按认证类型限制可见性（`claude-ai` / `console`） |

- **关键函数**: `getCommandName(cmd)`, `isCommandEnabled(cmd)`

### 3.3 hooks.ts -- 钩子类型

钩子系统的核心类型定义，含 Zod Schema 校验：

```ts
// 钩子回调类型
type HookCallback = {
  type: 'callback'
  callback: (input, toolUseID, abort, hookIndex?, context?) => Promise<HookJSONOutput>
}

// 聚合结果
type AggregatedHookResult = {
  permissionBehavior?: 'ask' | 'deny' | 'allow' | 'passthrough'
  additionalContexts?: string[]
  updatedInput?: Record<string, unknown>
  // ...
}
```

- `syncHookResponseSchema` 定义了按 `hookEventName` 区分的联合类型输出
- 编译期断言 `Assert<IsEqual<SchemaHookJSONOutput, HookJSONOutput>>` 确保 SDK 类型与 Zod 类型一致

### 3.4 permissions.ts -- 权限类型

权限系统的纯类型文件（无运行时依赖，打破循环引用）：

- **权限模式**: `default | acceptEdits | bypassPermissions | dontAsk | plan | auto | bubble`
- **权限行为**: `allow | deny | ask`
- **决策类型**: `PermissionAllowDecision | PermissionAskDecision | PermissionDenyDecision`
- **分类器**: `YoloClassifierResult` -- 自动模式下的 LLM 安全分类器结果（含两阶段 XML）
- **ToolPermissionContext**: 只读权限上下文，含规则映射和工作目录

### 3.5 logs.ts -- 日志与会话持久化

定义会话转录中所有可持久化的条目类型（`Entry` 联合类型）：

- `TranscriptMessage` -- 带父子关系的序列化消息
- `AttributionSnapshotMessage` -- Claude 贡献归因快照
- `ContextCollapseCommitEntry` / `SnapshotEntry` -- 上下文折叠（marble-origami）
- `WorktreeStateEntry` -- Git worktree 会话状态
- `ContentReplacementEntry` -- 内容替换记录（用于恢复）

### 3.6 plugin.ts -- 插件类型

插件加载与错误处理的完整类型系统：

- `LoadedPlugin` -- 已加载插件（含命令、代理、技能、钩子、MCP/LSP 服务器）
- `PluginError` -- 20+ 种结构化错误类型的判别联合
- `getPluginErrorMessage(error)` -- 错误消息格式化

### 3.7 generated/ -- Protobuf 生成类型

由 `protoc-gen-ts_proto` 自动生成：
- `claude_code_internal_event.ts` -- 内部遥测事件（环境元数据、Slack 上下文）
- `timestamp.ts` -- Google Protobuf Timestamp
- `auth.ts` -- 认证信息

## 4. src/schemas/ -- 数据校验 Schema

### hooks.ts -- 钩子配置 Schema

从 `settings/types.ts` 中提取以打破循环依赖。使用 Zod v4 + `lazySchema` 延迟初始化。

**四种钩子类型（判别联合，按 `type` 字段）**:

| 类型 | 字段 | 说明 |
|------|------|------|
| `command` | `command`, `shell`, `timeout`, `once`, `async`, `asyncRewake` | Shell 命令 |
| `prompt` | `prompt`, `model`, `timeout`, `once` | LLM 提示评估 |
| `http` | `url`, `headers`, `allowedEnvVars`, `timeout`, `once` | HTTP POST |
| `agent` | `prompt`, `model`, `timeout`, `once` | 代理验证器 |

**共享字段**: `if`（权限规则语法过滤）、`statusMessage`、`once`

**Schema 层次**:
```
HooksSchema (partialRecord)
  └─ HookMatcherSchema[] (matcher + hooks)
       └─ HookCommandSchema (discriminatedUnion by type)
```

- 导出推断类型: `HookCommand`, `BashCommandHook`, `PromptHook`, `AgentHook`, `HttpHook`
- `HooksSettings = Partial<Record<HookEvent, HookMatcher[]>>`

## 架构要点总结

1. **query 子模块拆分**: config/deps/tokenBudget/stopHooks 各司其职，config 为纯数据快照，deps 支持测试注入
2. **Context 层统一模式**: Provider + Hook + useSyncExternalStore，最小化渲染开销
3. **类型安全策略**: 品牌类型防混用、Zod Schema 与 TS 类型编译期对齐断言、判别联合覆盖错误路径
4. **循环依赖治理**: permissions.ts 和 schemas/hooks.ts 均为无运行时依赖的纯类型/Schema 提取
