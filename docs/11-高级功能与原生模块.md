# 11 - 高级功能与原生模块

本章分析 Claude Code 中五个独立子系统：KAIROS 助手会话历史、Buddy 伙伴系统、权限扩展桩、原生 TypeScript 替代实现，以及 CCR 上游代理。

---

## 1. src/assistant/ — KAIROS 助手会话历史

**文件**: `sessionHistory.ts`

### 功能定位

为 KAIROS 助手模式提供**远程会话事件分页拉取**能力。通过 OAuth 认证从 Anthropic API 读取历史会话消息。

### 核心 API

| 函数 | 作用 |
|------|------|
| `createHistoryAuthCtx(sessionId)` | 准备认证上下文（accessToken + orgUUID + headers），复用于多次分页请求 |
| `fetchLatestEvents(ctx, limit)` | 拉取最新一页事件（默认100条），使用 `anchor_to_latest` 参数 |
| `fetchOlderEvents(ctx, beforeId, limit)` | 基于游标 `before_id` 向前翻页 |

### 数据结构

```
HistoryPage {
  events: SDKMessage[]     // 按时间顺序
  firstId: string | null   // 本页最早事件 ID，用作下一页游标
  hasMore: boolean          // 是否还有更早的事件
}
```

### 设计要点

- 请求超时 15 秒，HTTP 错误静默降级（返回 null）
- 使用 `anthropic-beta: ccr-byoc-2025-07-29` 头标识 CCR BYOC 协议
- 分页大小常量 `HISTORY_PAGE_SIZE = 100`

## 2. src/buddy/ — Buddy 伙伴系统

**文件**: `types.ts`, `companion.ts`, `sprites.ts`, `prompt.ts`, `CompanionSprite.tsx`, `useBuddyNotification.tsx`

### 功能定位

一个**彩蛋级宠物伙伴系统**。用户可通过 `/buddy` 命令"孵化"一个ASCII小动物，它会在终端输入框旁边闲逛、偶尔冒出语言气泡。

### 架构分层

```
types.ts          → 物种/眼睛/帽子/稀有度/属性的类型定义
companion.ts      → 基于 userId 确定性生成伙伴骨骼（PRNG）
sprites.ts        → 18种物种的 ASCII 精灵图 + 帽子装饰
prompt.ts         → 向 LLM 注入伙伴上下文提示词
CompanionSprite.tsx → Ink 组件：动画循环、语言气泡、互动
useBuddyNotification.tsx → 启动时的彩虹提示
```

### 确定性生成机制 (companion.ts)

核心思路：**不存储骨骼，每次从 userId 重新生成**，防止用户篡改配置获得高稀有度。

1. 将 `userId + SALT("friend-2026-401")` 做 FNV-1a 或 Bun.hash 取 32 位种子
2. 用 Mulberry32 PRNG 依次抽取：稀有度 → 物种 → 眼睛 → 帽子 → 闪光 → 属性值
3. 结果缓存（单条，同 userId 命中直接返回）

**稀有度权重**: common:60, uncommon:25, rare:10, epic:4, legendary:1

**属性系统**: DEBUGGING / PATIENCE / CHAOS / WISDOM / SNARK 五项，各稀有度有不同下限，采用"一高一低其余散布"策略。

### 物种与精灵 (types.ts + sprites.ts)

18 种物种：duck, goose, blob, cat, dragon, octopus, owl, penguin, turtle, snail, ghost, axolotl, capybara, cactus, robot, rabbit, mushroom, chonk。

每种有 3 帧动画（5行x12列），用 `{E}` 占位符替换眼睛样式。帽子（crown/tophat/propeller 等）替换第 0 行。

> 注意：物种名用 `String.fromCharCode` 编码，避免触发构建产物中的模型代号敏感词检查。

### 提示词注入 (prompt.ts)

当伙伴存在且未静音时，向消息流注入 `companion_intro` 附件，告知 LLM："一只叫 {name} 的 {species} 坐在输入框旁边"，并指示 LLM 在用户直接与伙伴对话时保持低调。

### 发布节奏 (useBuddyNotification.tsx)

- 预告窗口：2026年4月1-7日（使用本地时间，非 UTC，实现跨时区持续传播）
- 正式上线：2026年4月起永久可用
- Anthropic 内部构建始终开启

## 3. src/moreright/ — 权限扩展（外部桩）

**文件**: `useMoreRight.tsx`

### 功能定位

这是一个**空桩（stub）**。真正的 `useMoreRight` Hook 仅存在于 Anthropic 内部构建中。

### 接口签名

```typescript
useMoreRight(args: {
  enabled: boolean
  setMessages, inputValue, setInputValue, setToolJSX
}) → {
  onBeforeQuery: (input, all, n) => Promise<boolean>
  onTurnComplete: (all, aborted) => Promise<void>
  render: () => null
}
```

外部版本所有回调都是空操作：`onBeforeQuery` 总返回 `true`，`onTurnComplete` 无动作，`render` 返回 `null`。

### 设计意图

通过 Hook 接口在查询前/轮次完成后注入扩展逻辑（如额外权限检查、UI 覆盖）。内部实现通过构建时的文件覆盖（overlay）替换此桩。文件注释明确说明"无相对导入"以确保外部桩可独立类型检查。

## 4. src/native-ts/ — 原生 TypeScript 实现

将原本依赖 Rust NAPI 原生模块的功能用纯 TypeScript 重写，消除平台编译依赖。

### 4.1 color-diff — 语法高亮差异渲染

**文件**: `color-diff/index.ts`

**替代对象**: `vendor/color-diff-src`（Rust，使用 syntect + bat + similar crate）

**TS 实现方案**:
- 语法高亮：highlight.js（延迟加载，避免 190+ 语法文件的 ~50MB/100-200ms 初始化代价）
- 文本差异：`diff` 包的 `diffArrays`
- ANSI 颜色：手写 truecolor/256色/16色 三模式渲染

**核心类**:
- `ColorDiff` — 接收 old/new 文本，输出带语法高亮的 unified diff
- `ColorFile` — 单文件语法高亮渲染
- `getSyntaxTheme()` — BAT_THEME 环境变量支持（当前为桩）

**与 Rust 版的差异**: highlight.js 语法覆盖不如 syntect，部分标识符和操作符无法着色。

### 4.2 file-index — 模糊文件搜索

**文件**: `file-index/index.ts`

**替代对象**: `vendor/file-index-src`（Rust，使用 nucleo 引擎）

**核心类**: `FileIndex`

| 方法 | 作用 |
|------|------|
| `loadFromFileList(paths)` | 去重 + 建索引（同步） |
| `loadFromFileListAsync(paths)` | 异步分片建索引，返回 `{queryable, done}` 两个 Promise |
| `search(query, limit)` | 模糊搜索，返回 Top-K 结果 |

**评分算法**（模拟 nucleo/fzf-v2）:
- 基础匹配 +16 分，边界/驼峰/连续各有加成
- 间隙惩罚：起始 -3，每字符 -1
- 智能大小写：全小写查询忽略大小写
- 包含 "test" 的路径降权 1.05x

**性能优化**:
- 26 位 bitmap 预过滤（O(1) 排除缺少查询字母的路径）
- Top-K 堆维护，避免全排序
- 异步构建每 4ms 让出事件循环
- `indexOf` 融合扫描：位置查找 + 间隙计算一趟完成

### 4.3 yoga-layout — Flexbox 布局引擎

**文件**: `yoga-layout/enums.ts`, `yoga-layout/index.ts`

**替代对象**: `yoga-layout`（Meta 的 C++ flexbox 引擎，通过 NAPI 绑定）

**实现范围**: 针对 Ink TUI 框架实际使用的 flexbox 子集，纯 TypeScript 重写：

- flex-direction（含 reverse）、flex-grow/shrink/basis
- align-items/self、justify-content（全部六种值）
- margin/padding/border/gap、width/height/min/max
- position: relative/absolute、display: flex/none/contents
- flex-wrap: wrap/wrap-reverse、align-content
- baseline 对齐、margin: auto

**未实现**: aspect-ratio、content-box、RTL 方向。

**API 完全兼容**: `Node` 类提供与 yoga-layout 相同的 setter/getter（`setWidth`, `setFlexGrow`, `calculateLayout` 等），`enums.ts` 值与上游一一对应。

## 5. src/upstreamproxy/ — CCR 上游代理

**文件**: `relay.ts`, `upstreamproxy.ts`

### 功能定位

在 CCR（Claude Code Remote）容器内运行时，为 agent 子进程（curl/gh/kubectl 等）提供透明 HTTPS 代理，自动注入组织级凭据（如 DD-API-KEY）。

### 架构概览

```
agent 子进程 ──CONNECT──► 本地 TCP relay ──WebSocket──► CCR 上游代理服务
                (127.0.0.1:N)                        (MITM TLS, 注入凭据)
                                                      ──► 真实上游服务
```

### relay.ts — CONNECT-over-WebSocket 中继

**为什么用 WebSocket**: CCR 入口是 GKE L7 路由，不支持原始 CONNECT。

**协议**: 客户端 TCP 字节封装为 `UpstreamProxyChunk` protobuf（手工编码，单字段 bytes，避免引入 protobufjs 依赖）。

**连接生命周期**:
1. 累积客户端 CONNECT 请求（等待 `\r\n\r\n`）
2. 打开 WebSocket 到 CCR 代理端点
3. 发送 CONNECT 行 + Proxy-Authorization
4. 双向透传数据，每 30 秒发送空 chunk 保活

**双运行时支持**:
- Bun: `Bun.listen()` + 手动处理 partial write 背压
- Node: `net.createServer()` + `ws` 包（预加载避免异步竞争）

**安全**:
- 最大 chunk 512KB（Envoy 缓冲上限）
- CONNECT 请求头上限 8KB
- 已建立隧道后不发送明文错误（避免破坏 TLS 流）

### upstreamproxy.ts — 容器侧初始化

**启动流程** (`initUpstreamProxy`):
1. 检查 `CLAUDE_CODE_REMOTE` 和 `CCR_UPSTREAM_PROXY_ENABLED` 环境变量
2. 从 `/run/ccr/session_token` 读取令牌
3. `prctl(PR_SET_DUMPABLE, 0)` 阻止 ptrace 读取堆内存中的令牌
4. 从 API 下载代理 CA 证书，与系统 CA 拼接写入 `~/.ccr/ca-bundle.crt`
5. 启动本地 relay
6. 删除令牌文件（令牌仅保留在进程堆中）

**环境变量注入** (`getUpstreamProxyEnv`):
- `HTTPS_PROXY` / `https_proxy` → `http://127.0.0.1:<port>`
- `NO_PROXY` → localhost/RFC1918/IMDS/anthropic.com/github.com/npm/pypi 等
- `SSL_CERT_FILE` / `NODE_EXTRA_CA_CERTS` / `REQUESTS_CA_BUNDLE` / `CURL_CA_BUNDLE` → CA 证书包路径

**设计原则**: 所有步骤 fail-open，代理故障不影响正常会话。子 CLI 进程通过继承父进程的环境变量复用已启动的 relay。

## 总结

| 模块 | 性质 | 关键技术 |
|------|------|----------|
